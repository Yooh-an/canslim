# Growth Stock Screener 아키텍처 설계

## 1. 시스템 개요

Growth Stock Screener는 SEC EDGAR API에서 제공하는 기업 재무 데이터를 기반으로 CAN SLIM 및 Minervini 성장주 기준에 부합하는 주식을 식별하는 파이썬 애플리케이션입니다. 이 문서는 시스템의 전반적인 구조, 주요 컴포넌트, 데이터 흐름 및 처리 과정을 설명합니다.

## 2. 아키텍처 다이어그램

+------------------+ +-------------------+ +-------------------+
 | | | | | | | SEC EDGAR API +---->+ Data Processing +---->+ Stock Screening +----> Results (CSV) | | | | | | +------------------+ +-------------------+ +-------+-----------+ | +---------------------+ | | | | | Market Data (YF) +<---+ | | +---------------------+



```

├── config/             # 설정 파일
├── data/              # 데이터 저장소

├── docs/              # 문서

├── requirements.txt   # 의존성 목록
└── README.md
```


## 3. 핵심 모듈 구성

### 3.1. 데이터 수집 모듈 (SEC API Client)
- **목적**: SEC EDGAR API에서 재무 데이터 수집
- **주요 기능**:
  - API 요청 관리 및 속도 제한 준수
  - 대량 제출 파일(submissions) 다운로드
  - XBRL 회사 사실(company facts) 데이터 다운로드
  - 로컬 저장소 관리
  - 다운로드 상태 추적 및 재개 기능

### 3.2. 데이터 처리 모듈
- **목적**: 원시 데이터를 구조화하고 필요한 재무 지표 계산
- **주요 기능**:
  - 제출 파일 파싱 및 회사 인덱스 생성
  - XBRL 데이터를 파싱하여 재무 지표 추출
  - 분기 및 연간 성장률 계산
  - 재무 비율 계산 (ROE, 이익률, 부채비율 등)
  - 데이터 정규화 및 구조화
  - Parquet 형식으로 효율적인 데이터 저장

### 3.3. 주식 스크리닝 모듈
- **목적**: 사용자 정의 기준에 따라 주식 필터링
- **주요 기능**:
  - 파싱된 데이터에서 성장 지표 기반 필터링
  - 주가 성과 데이터 통합 (yfinance 사용)
  - 기관 소유 데이터 통합 (선택적)
  - 스크리닝 결과 형식화 및 출력

### 3.4. 유틸리티 모듈
- **목적**: 공통 기능 및 지원 서비스 제공
- **주요 기능**:
  - 로깅 시스템
  - 설정 관리
  - 오류 처리
  - 성능 모니터링

## 4. 데이터 흐름

### 4.1. 다운로드 단계 (Mode: download)
1. 사용자가 `--mode download` 옵션으로 애플리케이션 실행
2. 설정 파일에서 SEC API 매개변수 로드
3. SEC EDGAR API에 연결하여 submissions 대량 파일 다운로드
4. 다운로드한 submissions 파일에서 활성 회사 목록 추출
5. 각 회사의 XBRL 회사 사실 파일 다운로드
6. 다운로드된 모든 데이터를 `data/raw` 디렉토리에 저장

### 4.2. 파싱 단계 (Mode: parse)
1. 사용자가 `--mode parse` 옵션으로 애플리케이션 실행
2. `data/raw` 디렉토리에서 원시 데이터 로드
3. submissions 파일에서 회사 메타데이터(CIK, 회사명, 티커 등) 추출
4. XBRL 회사 사실 파일에서 재무 데이터 추출
5. 필요한 재무 지표 및 성장률 계산
6. 데이터 유효성 검사 및 정규화
7. 처리된 데이터를 Parquet 형식으로 `data/processed` 디렉토리에 저장

### 4.3. 스크리닝 단계 (Mode: screen)
1. 사용자가 `--mode screen --config <config_file>` 옵션으로 애플리케이션 실행
2. 설정 파일에서 스크리닝 기준 로드
3. `data/processed` 디렉토리에서 Parquet 파일 로드
4. 기준에 따라 회사 필터링 (EPS 성장률, 매출 성장률, ROE 등)
5. yfinance API를 사용하여 통과한 회사의 주가 데이터 가져오기
6. S&P 500 대비 주가 성과 계산 및 필터링
7. 선택적으로 기관 소유 비율 데이터 통합
8. 최종 결과를 CSV 파일로 출력

## 5. 데이터 구조

### 5.1. 원시 데이터 구조
- **submissions.zip**: SEC 제출 내역 대량 파일 (JSON 형식)
- **company_facts/**: 개별 회사의 XBRL 사실 데이터 디렉토리
  - **CIK[회사ID].json**: 각 회사의 재무 데이터 (JSON 형식)

### 5.2. 처리된 데이터 구조
- **companies.parquet**: 처리된 회사 데이터 (Parquet 형식)
  - 스키마:
    - `cik`: 회사 식별 번호
    - `ticker`: 주식 티커 심볼
    - `name`: 회사명
    - `quarterly_eps_growth`: 분기별 EPS 성장률
    - `annual_eps_cagr`: 3년 연간 EPS CAGR
    - `revenue_growth`: 분기별 매출 성장률
    - `profit_margin`: 순이익률
    - `roe`: 자기자본수익률
    - `debt_to_equity`: 부채비율
    - 기타 재무 지표 및 메타데이터

### 5.3. 출력 데이터 구조
- **results.csv**: 스크리닝 결과 (CSV 형식)
  - 열:
    - 기본 회사 정보 (티커, 회사명)
    - 핵심 재무 지표
    - 주가 성과 데이터
    - 선택적 기관 소유 비율

## 6. 주요 알고리즘 및 로직

### 6.1. 성장률 계산
- 분기 EPS 성장률 = (최근 분기 EPS / 4분기 전 EPS) - 1
- 연간 EPS CAGR = ((최근 연간 EPS / 3년 전 EPS) ^ (1/3)) - 1
- 매출 성장률 = (최근 분기 매출 / 4분기 전 매출) - 1

### 6.2. 재무 비율 계산
- ROE = 순이익 / 자기자본
- 순이익률 = 순이익 / 매출
- 부채비율 = 총부채 / 자기자본

### 6.3. 스크리닝 로직
1. 1차 필터: 재무 지표 기준 적용
   - 분기 EPS 성장률 ≥ 설정값
   - 연간 EPS CAGR ≥ 설정값
   - 매출 성장률 ≥ 설정값
   - ROE ≥ 설정값
   - 이익률 ≥ 설정값
   - 부채비율 ≤ 설정값
2. 2차 필터: 주가 성과 기준 적용
   - 6개월 주가 상승률 > S&P 500 상승률
3. 선택적 3차 필터: 기관 소유 비율 기준 적용
   - 기관 소유 비율 ≥ 설정값

## 8. 의존성

- **외부 API**:
  - SEC EDGAR API: 재무 데이터 소스
  - yfinance: 주가 데이터 소스
  - 한국투자증권 KIS Open API(선택적): 단일 티커 분석용 해외주식 수정주가 일봉 보강
  - SimFin API(선택적): SEC 누락 재무지표 보강

- **주요 라이브러리**:
  - pandas: 데이터 처리 및 분석
  - pyarrow: Parquet 파일 처리
  - requests: API 통신
  - tqdm: 진행률 표시
